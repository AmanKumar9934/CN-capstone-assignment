# CN-capstone-assignment
Assignment 5: ACK-Based Communication Using ICMP Request/Reply and Extra Hops
Experiment Objectives
Students will understand acknowledgment-like behavior using ICMP echo request and echo
reply, then study how adding network hops affects response time and packet path.
Learning Outcomes
• Identify request/response behavior
• Observe hop impact on packet travel
• Compare direct vs multi-hop delay qualitatively
Concepts Used
• ICMP echo request/reply
• Multi-hop forwarding
• Simulation path and timing
Detailed Instructions
Task 1: Direct Setup
• PC1 — Switch — PC2
• Ping PC1 → PC2 in Simulation Mode
Task 2: Add Extra Hop
• Insert another switch between PC1 and PC2
• Repeat ping and observe additional forwarding steps
Task 3: Observation
• Write difference between both cases (steps/events/time)
Expected Output
• Packet path screenshots
• Pings successful in both setups
• Notes about extra hop effect
Practical Applications
• Network planning (latency impact)
• Understanding forwarding overhead
